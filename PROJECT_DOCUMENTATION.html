Part 1: Project Overview - Executive Summary

Project Identification

1. What is the domain of your project?

Answer: Cybersecurity, specifically Cyber Threat Intelligence (CTI) and Defensive Security Research using the MITRE ATT&CK Framework. The project bridges the gap between raw threat data and actionable intelligence through conversational AI.

Problem & Motivation

2. What is the problem statement?

Answer: Security analysts and defenders need quick, accurate information about adversarial techniques, tactics, detection strategies, and mitigation methods from the MITRE ATT&CK framework without navigating complex documentation or retrieving outdated information.

Answer: Build an AI-powered, defensively-focused conversational chatbot that:

  • Operates offline with local embeddings and models
  • Redirects malicious queries toward defensive research

Part 2: Paper Analysis - Academic Framework

Note: Your project is a practical system rather than a research paper. The following presents your project as if it were written as an academic publication.

1. What is the title of the paper?

Proposed Paper Title:
"CITRAG: Retrieval-Augmented Generation for Accessible, Grounded Cyber Threat Intelligence Question-Answering"

2. What is the main objective of the paper?

The paper proposes and evaluates an LLM-based question-answering system for cyber threat intelligence that grounds responses in authoritative MITRE ATT&CK documents, prevents hallucinations, and operates without cloud API dependencies.

3. What problem does the paper address?

Security analysts face cognitive overload when conducting threat intelligence research due to:

  • Manual navigation of 1200+ MITRE ATT&CK technique documents
  • Lack of semantic understanding in keyword-based searches
  • Need for rapid synthesis of information across techniques
  • Risk of misinformation from hallucinating language models
  • Privacy concerns with cloud-based CTI tools

9. What gap is identified in the paper?

Gap Statement:

"While retrieval-augmented generation has been successfully applied to open-domain QA, its application to specialized cybersecurity domains remains underexplored. Furthermore, no prior system simultaneously addresses: (a) semantic CTI retrieval, (b) LLM hallucination prevention, (c) defensive query reframing, and (d) offline operability—all critical requirements for enterprise CTI systems."

10. What is the novelty of the paper?

Novel Contributions:

  • First application of RAG to MITRE ATT&CK question-answering
  • Query rewriting layer redirecting offensive phrasing toward defensive research
  • Dual-LLM fallback strategy (Groq + Ollama) for cloud and offline operation
  • Hallucination guard validating answers against retrieved context
  • Privacy-preserving embeddings enabling airgapped deployment

Research Methodology

12. What are the main steps in the methodology?

  1. Data Preparation: Download MITRE ATT&CK STIX JSON, filter to relevant types
  2. Text Chunking: RecursiveCharacterTextSplitter (1000-char, 200-overlap)
  3. Embedding Generation: all-MiniLM-L6-v2 (384-dimensions)
  4. Index Creation: FAISS approximate nearest neighbor search
  5. Query Rewriting: Pattern matching for defensive framing
  6. Retrieval: Top-4 most similar chunks from FAISS
  7. LLM Generation: Groq API or Ollama fallback
  8. Hallucination Guard: Validate answer support from context
  9. Source Attribution: Return citations with response

31. What are limitations of the paper?

  • Limited scope (only MITRE ATT&CK; excludes CVEs, threat reports)
  • Offline data (manually downloaded STIX; no real-time feeds)
  • Single language (English only)
  • Hardcoded query rewriting (limited generalization)
  • No graph reasoning (cannot reason about technical relationships)
  • Smaller embedding model (potential semantic gaps)
  • No user study (lack of human evaluation)
